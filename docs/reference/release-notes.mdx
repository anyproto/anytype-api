A running, **calendar-dated** log of API changes as they ship in anytype-heart — including backward-compatible additions that do not introduce a new API version.

:::info[How this differs from the Changelog]

The [Changelog](./changelog) is per API version and only changes on breaking changes. These Release notes are per anytype-heart release, so backward-compatible additions appear by ship date instead of being backdated into an existing version.

:::

## May 13, 2026

Shipped in anytype-heart v0.50.5.  
Applies to API version [`2025-11-08`](./changelog#2025-11-08) (unchanged).

### Added

- File management endpoints — backward-compatible additions to API version `2025-11-08`:
  - `POST /v1/spaces/:space_id/files` to upload a file
  - `GET /v1/spaces/:space_id/files/:file_id` to download a file
  - `DELETE /v1/spaces/:space_id/files/:file_id` to delete a file

  Availability depends on the running anytype-heart build. A client can confirm support by checking for the `files` paths in the spec served at `GET /docs/openapi.json` by the local instance.

## December 11, 2025

Shipped in anytype-heart v0.46.3.  
Introduces API version [`2025-11-08`](./changelog#2025-11-08).

Markdown body patching, object type updates, inline tag creation, and property-based filtering.

### Added

- Markdown body patching via `UpdateObjectRequest`
- Update an object's type by patching `type_key`
- Inline tag creation in `CreatePropertyRequest` for `select` / `multi_select`
- Property-based filtering on listing and search endpoints

## June 17, 2025

Shipped in anytype-heart v0.41.0.  
Introduces API version [`2025-05-20`](./changelog#2025-05-20).

Challenge-based authentication, stable type/property keys, rate limiting, and built-in SVG icons. This version contains **breaking changes** — review them before upgrading a pinned client.

### Added

- New authentication endpoints: `POST /v1/auth/challenges`, `POST /v1/auth/api_keys`
- Stable, caller-specified type and property keys
- Built-in SVG type icons

## May 6, 2025

Shipped in anytype-heart v0.40.20.  
Introduces API version [`2025-04-22`](./changelog#2025-04-22).

First stable release carrying the `Anytype-Version` header. Full CRUD for spaces, types, properties, and tags. This version contains **breaking changes**.

### Added

- Space management: `PATCH /spaces/:space_id`
- Full type CRUD: `POST` / `PATCH` / `DELETE /spaces/:space_id/types[/:type_id]`
- Full property CRUD under `/spaces/:space_id/properties`
- Full tag CRUD under `/spaces/:space_id/properties/:property_id/tags`
